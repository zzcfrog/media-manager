# 时间线拖拽测试用例

基于项目 62（青海）。测 apply 重建的叙事/主旨时间联动是否正确。

## 测试原理

分镜时间点 = 「shot 顺序 + src 区间时长」累加（派生，不独立记录）。叙事时间 = 其内 shot 累加，主旨时间 = 其内叙事累加。拖拽改顺序/归属 → apply 重建 → 时间联动。

**预期规则**：
- 同叙事内重排：叙事/主旨时间**不变**（shots 集合不变，只顺序变）。
- 跨叙事移动：源叙事**减**（移出分镜时长）、目标叙事**增**、跨过的叙事**起始前移**（shots 不变但前面缩短）。
- 跨主旨移动：源主旨**减**、目标主旨**增**、跨过的主旨/叙事**起始前移**。
- 任何移动：总时长**不变**（分镜总数不变）。

## 项目 62 结构

```
act_1: nar_1_1[V0-62]   nar_1_2[V63-69]
act_2: nar_2_4[V70-77]  nar_2_5[V78-85]  nar_2_6[V86-94]  nar_2_7[V95-101]  nar_2_8[V102-112]  nar_2_9[V113-125]
act_3: nar_3_1[V126-132]  nar_3_2[V133-144]  nar_3_3[V145-153]  nar_3_4[V154-169]
```
VN = 第 N 个分镜（0-based，按 plan 全局顺序）。
